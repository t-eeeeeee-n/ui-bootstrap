SCREENS.md を作成または追記します。各画面のレイアウト・コンテンツ・コンポーネント使用を言語化します。

## パス解決

- SCREENS.md の保存先: `${UIB_DOCS_DIR:-.}/SCREENS.md`
- 参照: `${UIB_DOCS_DIR:-.}/PRODUCT.md`、`${UIB_DESIGN_DIR:-.}/DESIGN.md`

## 引数の解釈

- `$ARGUMENTS` に `--append` が含まれる場合: **追記モード**。既存 SCREENS.md を
  読み込んだ上で、新規画面のみを末尾に追加する
- 含まれない場合: 通常モード。新規作成または全面書き換え（既存あるなら確認を取る）

追記モードは複数ブランド並行運用で威力を発揮する：
「`/pricing` だけ後から追加したい」「Stripe 版で `/billing` を足したい」等。

## 前提の確認

1. `${UIB_DESIGN_DIR:-.}/DESIGN.md`（脱ブランド化済み）と
   `${UIB_DOCS_DIR:-.}/PRODUCT.md` が存在することを確認
2. 未完了の場合: `/ui-product-write` までの完了を案内して終了
3. 通常モードで `${UIB_DOCS_DIR:-.}/SCREENS.md` が既に存在する場合:
   「追記モード `/ui-screens-write --append` の方が安全では？」と確認

## ユーザーからヒアリングすること

### 通常モード
1. **画面一覧** — URL パス、ファイル名、画面の役割
2. **共通レイアウト** — 全画面共通の要素（サイドバー、上部バー等）
3. **画面ごとの具体仕様**
   - レイアウトの大枠
   - 主要セクション
   - 使うコンポーネント
   - 使うダミーデータ（PRODUCT.md を参照）
4. **最重要・最複雑画面の特定** — Step 5 のパイロット対象になる

### 追記モード
1. **追加する画面のリスト** — URL パスとファイル名
2. 各画面について:
   - 既存の共通レイアウトを踏襲するか（推奨）
   - 独自のレイアウトが必要な場合はその仕様
   - 使うコンポーネント、特に **既存 SCREENS.md にまだ無い新規コンポーネント**
3. **既存画面に影響する変更があるか**
   （例: settings.html のサブナビに新タブを追加する等）

## 実行

### 通常モード

以下の構造で `${UIB_DOCS_DIR:-.}/SCREENS.md` を作成し保存：

````markdown
# 画面仕様

## 共通レイアウト
[全画面共通の要素]

ダミーデータは PRODUCT.md の「ダミーデータ正典」を唯一の出典として使う。

## 1. [ファイル名].html  `[URL パス]`
- [レイアウトの大枠]
- [主要セクション]
- [使うコンポーネント]
- [ダミーデータの指定]

## 2. [ファイル名].html  `[URL パス]`
...

## [N]. [パイロット対象画面] 【パイロット対象・最複雑】
[特に詳細に]

## index.html（ナビゲーション）
- ルートに `index.html` を配置
- 全画面への直接リンクを一覧表示
- 各リンクの右に画面の目的を一行で
````

### 追記モード

1. 既存 SCREENS.md を読み込む
2. 既存画面の番号を把握（例: 1〜8 まである）
3. 新規画面を末尾に連番で追加（9, 10, ...）
4. 既存画面への変更が必要な場合は、該当セクション直下に
   「## X. [既存画面名] 更新」という形で差分を書く
5. index.html セクションも更新して新画面を追記

## 書き方のコツ

以下はユーザーに作業プロセス中に伝える指針:

- **一画面 8〜15 行** を目安に
- レイアウトの記述は **上から下、左から右** の順で
- 「このダミーデータを使う」を **明示** する（Claude Code の裁量を減らす）
- 最重要画面には **【パイロット対象】** マーカーを必ず付ける
- 追記モードでは、既存画面との整合性（例: サブナビの項目追加）を忘れずに記述

## 次のステップ

### 通常モードの完了後
「Step 4 完了。3 点セット（DESIGN.md / PRODUCT.md / SCREENS.md）が揃いました。
次は `/ui-mock-pilot` で最複雑画面 1 枚を先行生成し、共有 CSS 資産を作ります。」

### 追記モードの完了後
「SCREENS.md に [N] 画面を追記しました。
既に mock を生成済みの場合、新規画面だけ追加生成するには:
`/ui-mock-pilot [新規画面名]` を実行、
その後 `/ui-mock-rest` で不足分を埋めてください。」
