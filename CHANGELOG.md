# Changelog

## v2.0.0 — 2026-04-23

### 追加

- **`/ui-set-paths` コマンド** — セッション中の入出力パスを環境変数で制御
- **複数ブランド並行運用サポート** — 同一プロダクトを複数のブランドで作り分け可能に
- **`/ui-screens-write --append`** — 既存 SCREENS.md への追記モード
- **環境変数サポート** — `UIB_DOCS_DIR` / `UIB_DESIGN_DIR` / `UIB_MOCK_DIR`
- **WORKFLOW.md に複数ブランド運用セクション** を追加
- **トラブルシューティング** セクションを拡充

### 変更

- 全 slash command がパスを環境変数ベースで解決するように変更
  （未設定時は従来通りカレントディレクトリ基準、後方互換あり）
- `/ui-bootstrap` オーケストレータにパス設定の確認ステップを追加
- 推奨ブランド表に `stripe` と `vercel` を追加（エンタープライズ系の選択肢）

### 移行ガイド（v1 → v2）

v1 から v2 への移行は **無変更で動作する** 設計です。以下はすべて任意：

1. `.claude/commands/` 配下を v2 のファイルで上書き
2. `WORKFLOW.md` を v2 で上書き
3. 既存プロジェクトは環境変数未設定のまま動作（v1 互換）
4. 新規に複数ブランド運用したい場合のみ `/ui-set-paths` を使う

### 既知の制限

- 環境変数は Claude Code セッション単位でスコープされる。
  セッション再起動後は `/ui-set-paths` を再実行する必要がある
- SKILL.md 側は現状 v1 相当の記述のまま。次のマイナー版で追従予定

## v1.0.0 — 初版

- WORKFLOW.md をソースオブトゥルースとして定義
- SKILL.md による自動発火サポート
