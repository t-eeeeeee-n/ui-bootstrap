SCREENS.md を作成します。各画面のレイアウト・コンテンツ・コンポーネント使用を言語化します。

## 前提の確認

1. DESIGN.md（脱ブランド化済み）と PRODUCT.md が存在することを確認
2. 未完了の場合: `/ui-product-write` までの完了を案内して終了

## ユーザーからヒアリングすること

1. **画面一覧** — URL パス、ファイル名、画面の役割
2. **共通レイアウト** — 全画面共通の要素（サイドバー、上部バー等）
3. **画面ごとの具体仕様**
   - レイアウトの大枠
   - 主要セクション
   - 使うコンポーネント
   - 使うダミーデータ（PRODUCT.md を参照）
4. **最重要・最複雑画面の特定** — Step 5 のパイロット対象になる。後続の
   量産に必要なコンポーネントをほぼカバーする画面を選ぶ

## 実行

以下の構造で SCREENS.md を作成し保存：

````markdown
# 画面仕様

## 共通レイアウト
[全画面共通の要素]

ダミーデータは PRODUCT.md の「ダミーデータ正典」を唯一の出典として使う。

## 1. [ファイル名].html  `[URL パス]`
- [レイアウトの大枠]
- [主要セクション]
- [使うコンポーネント]
- [ダミーデータの指定]

## 2. [ファイル名].html  `[URL パス]`
...

## [N]. [パイロット対象画面] 【パイロット対象・最複雑】
[特に詳細に]

## index.html（ナビゲーション）
- ルートに `index.html` を配置
- 全画面への直接リンクを一覧表示
- 各リンクの右に画面の目的を一行で
````

## 書き方のコツ

以下はユーザーに作業プロセス中に伝える指針:

- **一画面 8〜15 行** を目安に
- レイアウトの記述は **上から下、左から右** の順で
- 「このダミーデータを使う」を **明示** する（Claude Code の裁量を減らす）
- 最重要画面には **【パイロット対象】** マーカーを必ず付ける
